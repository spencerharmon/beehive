honeybee performance ✅/🐝

Derived on read from git — nothing stored, nothing to drift. delivered tasks (✅) = tasks marked [DONE] (a task can take more than one merge, so we count the task) · honeybees (🐝) = one honeybee pass · ✅/🐝 = delivered tasks per honeybee · stranded = tasks with finished work on a pushed branch that never merged (recoverable — GC never drops an unmerged branch). Expand by model to compare agent models head-to-head (delivered is attributed to the model of a done task's most-recent session). Expand deliveries to trace a DONE task to (a) the hive commit that flipped its PLAN status and (b) its submodule change doc — "why is this DONE" in one click across both repos. See docs/conflict-resolution.md.

filter & group by

Chips are ANDed (every filter must match) — no query language, just composable tag=value chips. Group by one or more tag keys to compare models, kinds, submodules, or any config-declared tag side by side. State lives in the URL, so a filtered/grouped view is shareable and bookmarkable. An unknown tag key or value simply yields no rows, never an error.
